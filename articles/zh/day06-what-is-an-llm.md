# 第六天：大语言模型到底是什么？

> **核心问题**：大语言模型（Large Language Model，LLM）在生成文本时究竟在做什么——为什么这个看似简单的机制能产生智能的感觉？

---

## 开篇：世界上最复杂的自动补全

你用过手机键盘的自动补全功能。输入"生日快"，它推荐"乐"；输入"我马上"，它推荐"来"。这个功能有点用，偶尔令人惊喜，但没有人会称它为智能。

现在想象把这个自动补全放大十亿倍。不只是训练在你个人的消息上，而是训练在人类几乎所有的文字上——小说、科学论文、Reddit 争论、Stack Overflow 的答案、莎士比亚……以及关于莎士比亚的 Reddit 争论。而且不是简单的查找表，而是一个拥有数千亿可调参数的系统，在数千块 GPU 上训练了数月，学习语言中错综复杂的统计规律。

结果就是大语言模型（Large Language Model，LLM）。它的核心工作依然是"预测下一个词"。但这个看似简单的目标，在大规模数据上应用时，产生了能写代码、解释量子力学、翻译语言、通过执照考试的系统。

这是现代 AI 最令人惊讶的发现之一：**你不需要显式编程出智能。只需要在足够多的数据上大规模地预测下一个词元（Token）即可。**

---

## 1. 一句话定义

大语言模型是一个计算以下概率的函数：

$$
P(\text{next token} \mid \text{context})
$$

就这么简单。给定一串词元（单词、子词、标点），它输出下一个词元最可能是什么的概率分布。

其他一切——看似智能的回答、代码生成、多步推理——都是将这个单一操作一次一个词元地反复应用的结果。

![图 1：LLM 作为下一词元概率预测机器](./images/day06/next-token-prediction.png)

## 2. 定义背后的数学

### 2.1 通过链式法则（Chain Rule）计算联合概率

语言模型如何给一整个句子赋予概率？通过**概率的链式法则（Chain Rule of Probability）**：

$$
P(x_1, x_2, \ldots, x_T) = \prod_{t=1}^{T} P(x_t \mid x_1, x_2, \ldots, x_{t-1})
$$

这意味着：一个句子的概率等于句子中每个词在其前面所有词给定下的条件概率的乘积。LLM 学习计算这些条件概率。

以句子 "The cat sat on the mat"（猫坐在垫子上）为例：

$$
\begin{aligned}
P(\text{"The cat sat"}) &= P(\text{"The"}) \\
&\times P(\text{"cat"} \mid \text{"The"}) \\
&\times P(\text{"sat"} \mid \text{"The cat"})
\end{aligned}
$$

### 2.2 训练：从预测错误中学习

模型通过最小化**交叉熵损失（Cross-Entropy Loss）**进行训练——每当模型给正确的下一个词元分配了较低概率时，就对其进行惩罚：

$$
\mathcal{L} = -\frac{1}{T} \sum_{t=1}^{T} \log P(x_t \mid x_1, \ldots, x_{t-1})
$$

如果模型对正确下一词元预测了 10% 的概率，损失为 $-\log(0.1) \approx 2.30$；如果预测了 90%，损失为 $-\log(0.9) \approx 0.10$。在数万亿个样本上，训练推动模型对正确答案尽可能自信。

### 2.3 自回归生成（Autoregressive Generation）

在推理时，生成是**自回归（Autoregressive）**的：模型生成一个词元，将其追加到上下文中，再用这个扩展后的上下文预测下一个词元，如此循环：

$$
x_{T+1} \sim P(\cdot \mid x_1, \ldots, x_T)
$$
$$
x_{T+2} \sim P(\cdot \mid x_1, \ldots, x_T, x_{T+1})
$$
$$
\vdots
$$

这就是为什么生成感觉流畅——每个新词元都是在与前面所有内容保持连贯的条件下选出的。

## 3. "大"是什么意思？

"大语言模型"中的"大"字承载了很多含义。它指三个相互关联的方面：

![图 2：大语言模型中的"大"](./images/day06/what-makes-llm-large.png)
*图 2：面板 (a) 展示了从 BERT（2018 年 3.4 亿参数）到现代超过 4000 亿参数模型的规模爆炸。面板 (b) 总结了规模的关键维度。面板 (c) 展示了典型的训练数据组成——互联网（Common Crawl）占主导，辅以书籍、代码和结构化知识。*

### 3.1 参数（Parameters）：习得的知识

现代 LLM 有数百亿到数千亿个**参数（Parameters）**——矩阵中的数值，编码了模型的知识和能力。把参数想象成庞大关联网络的权重："当我看到词元序列'埃菲尔铁塔'时，'巴黎'应该获得高概率。"这些权重在训练时被调整以最小化预测误差。

GPT-3 有 1750 亿参数；GPT-4 估计约 1.76 万亿。作为参考，人类大脑大约有 100 万亿个突触——但突触和参数不能直接比较（模型通常在单位计算量上效率更高）。

### 3.2 训练数据：压缩的知识

LLM 在 **1 万亿到 15 万亿词元**的文本上训练。这大约相当于把国会图书馆里的每一本书读数百遍，再加上消费互联网上相当大比例的文本。

一个类比：把训练后的模型想象成**人类书面知识的压缩表示**。权重并不是字面上存储了句子——而是编码了深层且错综复杂的统计模式，模型能从中重构或泛化。这就像把你见过的每张照片都存下来，和把识别人脸的能力学会足以认出从未见过的脸——两者的本质差异。

### 3.3 算力（Compute）：工业化过程

训练前沿 LLM 需要数万块专用 GPU 并行运行数月。GPT-3 的训练估计仅算力成本就高达 400-500 万美元；训练 GPT-4 据报道花费超过 1 亿美元。这不只是"跑代码"——而是需要精心设计数据管道、分布式训练、数值稳定性和硬件故障恢复的工业规模优化过程。

## 4. Transformer：让一切成为可能的架构

LLM 的"是什么"是*下一词元预测*，而"怎么做到的"几乎无一例外是 **Transformer 架构**（我们在第 3 天和第 4 天探索过）。简要回顾一下为什么：

在 Transformer 之前，序列模型是循环神经网络（RNN）——逐个处理词元，维护一个必须编码所有相关信息的隐藏状态。这是个瓶颈：处理第 500 个词元时，模型几乎已"遗忘"第 1 个。

Transformer 用**自注意力（Self-Attention）**取代了这一机制：每个词元都能直接"看到"上下文窗口中的其他所有词元，由学习到的注意力权重决定什么是相关的。这意味着：
- 无遗忘——整个上下文始终可访问
- 可并行化——训练时所有词元可同时处理
- 可扩展——更大的模型只需更多注意力头和更大的权重矩阵

在第 6 天，这样理解就够了：Transformer 是引擎；下一词元预测是燃料；规模让引擎轰鸣。

## 5. 从下一词元预测到表观智能

这是很多人感到困惑的地方。"预测下一个词"怎么能产生能做以下事情的系统：
- 写出可运行的 Python 代码
- 解释法国大革命
- 调试段错误（Segfault）
- 通过律师资格考试

答案还未被完全理解，但目前最佳假说是**压缩即理解**。为了准确预测蛋白质折叠研究论文中的下一个词元，模型必须已经学到了关于蛋白质折叠的某些内容。为了预测 Python 函数中的下一个词元，它必须理解 Python 语法。为了预测小说中的对话，它必须对人类心理有所建模。

训练目标*迫使*模型发展出能捕捉数据底层结构的内部表示。这种广度和深度的预测需要构建某种类似世界模型的东西。

## 6. 推理流水线（Inference Pipeline）实践

让我们追踪一下你向 LLM 发送消息时究竟发生了什么：

![图 3：LLM 推理流水线](./images/day06/llm-inference-pipeline.png)
*图 3：面板 (a) 展示从原始文本到生成词元的完整推理流水线——经过分词、嵌入、Transformer 层和语言模型头（Language Model Head）。面板 (b) 展示温度参数如何控制输出概率分布的尖锐程度，在确定性和多样性之间权衡。*

1. **分词（Tokenization）**：你的文本被拆分成词元（子词，英文大约每个词元 3-4 个字符）。词汇表通常包含 5 万到 10 万个词元。

2. **嵌入（Embedding）**：每个词元被映射到一个稠密向量（如 768 或 4096 维）。含义相似的词元在这个空间中位置相近——"猫"和"小猫"会在这个空间中彼此靠近。

3. **Transformer 处理**：词元向量流经数十层（或数百层）Transformer 层。自注意力让每个词元通过关注其他词元来更新自身表示。到最后一层时，每个词元的向量编码了丰富的上下文信息。

4. **语言模型头（Language Model Head）**：最后一个词元的向量通过一个权重矩阵投影，为词汇表中每个词元产生一个分数——50,000+ 个数字。

5. **Softmax**：分数通过 Softmax 转换为概率。分数最高的词元获得最高概率。

6. **采样（Sampling）**：根据这些概率选择一个词元。**温度（Temperature）= 0**（贪婪解码）时，始终选最高概率的词元。**温度 > 1** 时，分布被"拉平"，产生更多样（有时更有创意，有时更错误）的输出。

## 7. 代码示例：模拟核心思想

这是一个简化的 PyTorch 代码示例，展示 LLM 核心机制——不是完整实现，但足以看清数学原理：

```python
import torch
import torch.nn.functional as F

def next_token_probabilities(logits: torch.Tensor, temperature: float = 1.0) -> torch.Tensor:
    """
    将原始 logits 转换为下一词元的概率分布。
    
    Args:
        logits: 模型最后一层的原始分数，形状 (vocab_size,)
        temperature: 控制尖锐程度。
                     < 1.0 = 更确定性, > 1.0 = 更随机
    Returns:
        词汇表上的概率分布，形状 (vocab_size,)
    """
    # 用温度缩放 logits
    scaled_logits = logits / temperature
    
    # 通过 softmax 转换为概率
    # softmax(x_i) = exp(x_i) / sum(exp(x_j) for all j)
    probs = F.softmax(scaled_logits, dim=-1)
    
    return probs


def greedy_decode(probs: torch.Tensor) -> int:
    """始终选择概率最高的词元。"""
    return torch.argmax(probs).item()


def sample_decode(probs: torch.Tensor) -> int:
    """按概率分布采样词元。"""
    return torch.multinomial(probs, num_samples=1).item()


# 示例：假设词汇表大小为 5
# 词元: ["巴黎", "伦敦", "柏林", "里昂", "其他"]
logits = torch.tensor([3.5, 1.2, 0.8, 0.3, -0.5])

# 不同温度，不同行为
for temp in [0.3, 1.0, 2.0]:
    probs = next_token_probabilities(logits, temperature=temp)
    print(f"T={temp}: 巴黎={probs[0]:.3f}, 伦敦={probs[1]:.3f}, 柏林={probs[2]:.3f}")

# 输出:
# T=0.3: 巴黎=0.986, 伦敦=0.010, 柏林=0.003
# T=1.0: 巴黎=0.850, 伦敦=0.099, 柏林=0.066
# T=2.0: 巴黎=0.565, 伦敦=0.170, 柏林=0.134
```

关键洞察：在低温度下，模型非常自信，几乎总是选"巴黎"。在高温度下，它可能偶尔选"柏林"——这对事实性问题来说可能是错的，但对生成多样化创意文本来说可能是期望的行为。

## 8. 常见误解

### ❌ "LLM 像人类一样理解语言"

LLM 在*预测*语言模式方面极为出色。这在哲学意义上是否构成"理解"，存在真正的争议。有一点是明确的：LLM 没有感官体验，没有具身性，默认情况下跨对话也没有持久记忆。它们的"知识"在训练时就固定了。它们可能根本不"理解"任何东西——但它们学到的统计模式足够深层，因而极为实用。

### ❌ "LLM 只是查找表/搜索引擎"

查找表存储的是明确的答案。LLM 根据学习到的*模式*逐词元生成回答，而非存储的文本。这就是为什么它能写出一首从未显式见过主题的诗，或以新颖的方式组合概念。它更像一位内化了乐理的爵士乐手，而非一位播放现有录音的 DJ。

### ❌ "越大越好"

虽然规模带来了显著的改进，但存在收益递减。如果经过更好微调，运行在你笔记本电脑上的 70 亿参数模型可能在特定窄任务上优于 1750 亿参数模型。Chinchilla 缩放定律（第 9 天）表明 GPT-3 实际上*训练不足*——用更多数据、更少参数，可以胜过更多参数、更少数据的方案。

### ❌ "模型知道自己知道什么"

LLM 没有可靠的机制来判断自己知道什么、不知道什么。这是幻觉（Hallucination）的根本原因——我们将在第 21 天深入探讨。即使训练分布中不存在正确答案，模型也会自信地产生一个看起来合理的下一词元。

## 9. 为什么"大"是关键洞察

多年来，研究者构建语言模型，但它们始终局限于窄范围。它们能相当好地预测报纸语料库中的下一个词，但无法泛化。来自规模研究的关键洞察是：

**在规模阈值上出现了无法从更小模型预测的涌现（Emergent）能力。**

一个 1000 万参数的模型难以完成上下文学习（In-Context Learning）；一个 1000 亿参数的模型却出人意料地擅长——即使没有人显式编程这种能力。这就是**涌现（Emergence）**现象（第 10 天），也是为什么前沿 AI 实验室如此大力投入算力：他们实际上是在购买无法通过更聪明的工程获得的能力。

## 10. 能力边界与学术争议

理解了 LLM *是什么*之后，同样重要的是理解它*不是*什么。LLM 很强大，但它不是万能药——AI 研究界正在激烈辩论它的根本局限性。

### 10.1 LLM 擅长什么

| 能力 | 为什么有效 |
|------|-----------|
| **文本生成** | 直接训练目标 |
| **翻译** | 海量多语言数据 |
| **摘要** | 压缩隐含在下一词元预测中 |
| **代码生成** | 代码遵循可学习的模式 |
| **风格转换** | 训练数据中的风格模式 |
| **少样本学习** | 涌现的上下文学习能力 |

这些任务有一个共同点：它们可以被框定为输入序列的*合理延续*。LLM 本质上是*什么听起来对*的大师，基于它们吸收的模式。

### 10.2 LLM 的短板（以及正在改进的部分）

| 局限性 | 根本原因 | 2026 现状 |
|--------|---------|-----------|
| **精确计算** | 模式匹配，不是计算 | ✅ **基本解决**（CoT + 验证） |
| **可靠的事实检索** | 知识是压缩的，不是索引化的 | 🟡 RAG 改善中 |
| **因果推理** | 相关性 ≠ 因果 | ✅ **大幅改善**（CoT/ToT） |
| **实时学习** | 推理时权重冻结 | ❌ 仍未解决 |
| **自我认知** | 没有可靠的不确定性机制 | 🟡 校准改善中 |
| **长程规划** | 短视的词元生成 | 🟡 Agent 改善中 |

**2026 年的现实**：思维链（Chain-of-Thought, CoT）和推理模型（如 o1/o3）已经大幅提升了数学和推理能力。模型现在能解 IMO 级别的数学题，通过律师资格考试。然而，这是通过*学习逐步推理*而非根本性的架构改变实现的——关于这是"真正的推理"还是"对推理模式的精妙模式匹配"，辩论仍在继续。

**仍然困难的问题**：实时学习（不重新训练就更新知识）、可靠的自我认知（知道何时说"我不知道"）、以及需要真实世界交互而非纯文本操作的任务。

### 10.3 "随机鹦鹉"批评

2021 年，Bender 等人的著名论文引入了**"随机鹦鹉"（Stochastic Parrot）**这个术语来描述 LLM：

> "随机鹦鹉是一个随意拼接语言形式序列的系统……没有任何对意义的引用。"

**批评观点**：LLM 操纵符号，但没有与现实世界的联系。它们学习词语之间的相关性，但不是这些词语*指代*的概念。一个孩子通过触摸炉子学会"烫"；LLM 通过观察哪些词出现在"烫"附近来学习。

**反驳观点**：在足够的规模下，足够丰富的相关性是否*本身就成为*一种理解？辩论仍在继续。

### 10.4 Yann LeCun 与世界模型论点

**Yann LeCun**，前 Meta 首席 AI 科学家（现已离职创立自己的 AI 研究公司），是 LLM 范式最直言不讳的批评者之一。他的核心论点：

1. **LLM 缺乏世界模型**：它们不构建世界如何运作的内部表征——只建模文本分布。

2. **下一词元预测是不够的**：人类认知涉及预测世界的未来状态，而不只是下一个词。

3. **LLM 无法推理**：它们可以模仿见过的推理模式，但不进行真正的因果推断。

LeCun 倡导一种不同的方法：**联合嵌入预测架构（JEPA, Joint Embedding Predictive Architecture）**——在*表征空间*而非词元空间进行预测，并构建分层世界模型。

### 10.5 正在探索的替代架构

| 架构 | 核心思想 | 现状 |
|------|---------|------|
| **JEPA**（LeCun） | 预测表征而非词元；构建世界模型 | 研究阶段 |
| **V-JEPA** | 基于视频的 JEPA，学习物理规律 | 2024 年发表 |
| **Mamba / 状态空间模型** | O(n) 复杂度处理超长序列（100K+ tokens）；短序列场景 Transformer 仍更强 | 在长上下文场景逐渐普及 |
| **神经符号 AI** | 结合神经网络与符号推理 | 活跃研究 |
| **检索增强生成（RAG）** | 用外部知识给 LLM 接地 | 生产应用 |

**诚实的答案**：截至 2026 年，没有任何替代方案匹配 LLM 的通用能力，但 LeCun 等人指出的根本局限性仍未解决。

### 10.6 平衡的视角

LLM 最好被理解为：
- ✅ **在语言中心任务上极其强大**
- ✅ **作为大系统中的有用组件**（配合工具、检索、验证）
- ❌ **不是 AGI**——缺乏真正的推理、世界模型和自我意识
- ❌ **在需要事实精确性的任务上不可靠**（除非有验证）

**实用结论**：用 LLM 做它擅长的事，为它不擅长的事建立护栏，并持续关注领域发展。

## 13. 总结

| 概念 | 一句话解释 |
|------|-----------|
| LLM 核心操作 | $P(\text{next token} \mid \text{context})$ — 预测下一个词元 |
| 链式法则 | $P(\text{sentence}) = \prod_t P(x_t \mid x_{<t})$ |
| 训练目标 | 在下一词元预测上最小化交叉熵损失 |
| "大" | 指参数（1000 亿+）、训练数据（1 万亿+ 词元）和算力 |
| 自回归生成 | 一次生成一个词元，将输出反馈为输入 |
| 温度 | 控制输出分布尖锐程度；越低越确定性 |
| 涌现 | 在规模阈值上出现的能力，在更小模型中不可见 |
| 幻觉的根本原因 | 没有机制区分"我知道这个"和"我预测一个合理的内容" |

**核心结论**：大语言模型是一个计算 $P(\text{next token} \mid \text{context})$ 的函数，在大规模数据上训练以最小化对人类文本的预测误差。我们观察到的智能并非显式编程——它从准确预测人类生成的海量多样文本的压力中*涌现*出来。你见过的 LLM 所做的一切（代码、推理、翻译、创意）都是这个看似简单的目标在前所未有的规模上应用的结果。
